# 损失函数优化文档

## 概述

本文档描述了对前列腺癌多模态分割项目损失函数的优化工作。通过引入多种损失函数和组合策略，我们旨在提高模型在前列腺分割任务上的性能。

## 优化内容

### 1. 新增损失函数

#### 1.1 Focal Loss
- **目的**：解决类别不平衡问题，特别适用于前景（前列腺）区域相对较小的情况
- **实现**：基于标准的Focal Loss，通过调整gamma参数控制对难分类样本的关注程度
- **参数**：
  - `gamma`: 聚焦参数，默认为2.0

#### 1.2 Boundary Loss
- **目的**：提高分割边界的准确性，减少边界模糊
- **实现**：基于距离变换的边界损失，关注分割边界与真实边界之间的差异
- **特点**：对边界像素赋予更高权重

#### 1.3 Combined Loss
- **目的**：结合多种损失函数的优势，平衡不同方面的优化目标
- **实现**：加权组合Dice Loss、Focal Loss和Boundary Loss
- **参数**：
  - `dice_weight`: Dice Loss权重，默认为0.5
  - `focal_weight`: Focal Loss权重，默认为0.4
  - `boundary_weight`: Boundary Loss权重，默认为0.1

### 2. 配置文件更新

在`config.yml`中添加了损失函数配置选项：

```yaml
# 损失函数配置
# 可选类型: 'dice', 'focal', 'boundary', 'combined'
loss:
  type: 'combined'  # 使用组合损失函数
  # Focal损失参数
  gamma: 2.0
  # 组合损失权重 [dice, focal, boundary]
  weights: [0.5, 0.4, 0.1]
```

### 3. 训练代码更新

#### 3.1 损失函数选择
- 根据配置文件动态选择损失函数
- 支持四种损失函数类型：'dice', 'focal', 'boundary', 'combined'

#### 3.2 训练监控增强
- 对于组合损失，记录并显示各组件的值
- 在TensorBoard中添加各损失组件的曲线
- 在CSV日志中保存各损失组件的详细记录

#### 3.3 输出信息优化
- 训练过程中显示各损失组件的值
- 便于分析训练过程中各损失组件的变化趋势

## 使用方法

### 1. 使用单一损失函数

#### Dice Loss
```yaml
loss:
  type: 'dice'
```

#### Focal Loss
```yaml
loss:
  type: 'focal'
  gamma: 2.0  # 可选，默认为2.0
```

#### Boundary Loss
```yaml
loss:
  type: 'boundary'
```

### 2. 使用组合损失函数（推荐）
```yaml
loss:
  type: 'combined'
  weights: [0.5, 0.4, 0.1]  # [dice, focal, boundary]
```

## 预期效果

1. **分割精度提升**：通过组合多种损失函数，模型能够同时关注区域重叠、难分类样本和边界精度
2. **边界质量改善**：Boundary Loss的引入有助于提高前列腺边界的分割准确性
3. **训练稳定性**：Focal Loss有助于缓解类别不平衡问题，提高训练稳定性
4. **可解释性增强**：通过监控各损失组件，更好地理解模型的学习过程

## 实验建议

1. **基线对比**：首先使用原始的Dice Loss作为基线
2. **单一损失测试**：分别测试Focal Loss和Boundary Loss的效果
3. **组合损失调优**：调整组合损失中各组件的权重，找到最佳组合
4. **参数敏感性分析**：测试不同gamma值对Focal Loss的影响

## 注意事项

1. **计算开销**：Boundary Loss需要计算距离变换，可能会增加一定的计算开销
2. **权重平衡**：组合损失中的权重需要根据具体任务进行调整，不平衡的权重可能导致训练不稳定
3. **收敛速度**：不同的损失函数可能导致不同的收敛速度，需要相应调整学习率和训练策略
